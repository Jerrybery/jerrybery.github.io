Orca Hand Replica

A low-cost replica of the open-source ORCA Hand, rebuilt with Feetech servos and 3D-printed parts. By aligning the control protocol, register map, and high-level API to the original Dynamixel-based design, the hand achieves near-equivalent functionality at roughly a tenth of the cost.

ORCA Hand Feetech servos API-compatible control

Overview

With a classmate, I rebuilt the open-source ORCA Hand from scratch using Feetech servos instead of the original Dynamixel actuators. We redesigned the finger and palm structure for 3D printing, adapted the control software to Feetech's protocol, and aligned the register addresses and Python APIs so existing ORCA Hand scripts keep working with minimal changes.

The result is a fully articulated dexterous hand that can connect, enable torque, set joint positions, and run smooth interpolated motions — all through the same high-level OrcaHand interface as the upstream project, but at a much lower hardware cost.

Key Features

Feetech servo adaptation

Replaced Dynamixel actuators with Feetech servos while preserving joint ranges, torque behavior, and control semantics.

API-compatible hardware layer

Mapped Feetech registers and addresses to the original Dynamixel-style interface so existing scripts and tools remain usable.

Smooth interpolated motion

set_joint_pos splits large joint moves into small steps, producing natural grasp and release motions.

Calibration & tensioning workflow

Dedicated scripts for tensioning tendons, calibrating joint limits, moving to neutral, and zeroing the hand before operation.

FastAPI control server

An HTTP interface exposes config read/write, motor control, and joint-position commands for remote or programmatic use.

Offline testing with mock hardware

MockDynamixelClient lets the full control stack run and be tested without a physical hand connected.

Implementation Highlights

  • Protocol alignment: Feetech register map and packet format are adapted to match the original Dynamixel SDK calls, keeping the public API unchanged.
  • Model-driven configuration: Joint limits, motor IDs, and calibration offsets live in config.yaml and calibration.yaml per hand model.
  • Unified hardware abstraction: OrcaHand handles connect, torque enable, and joint position commands independently of the underlying servo brand.
  • Interpolated joint control: Multi-joint targets are interpolated step-by-step to avoid sudden tendon loads and jerky motion.
  • CI-ready without hardware: MockDynamixelClient plus pytest lets the core logic be validated in GitHub Actions.
