HMI Manipulation Demo

A closed-loop simulation of a UR5e robotic arm with a Robotiq 2F-85 gripper in MuJoCo, controlled through real-time MediaPipe hand-gesture arbitration from a webcam.

UR5e + Robotiq 2F-85 Open-palm / closed-fist Claude Code built
0:00

Overview

This demo runs entirely on a single macOS laptop: a physics-backed UR5e arm tracks a circular trajectory above a table while a webcam watches your hand. An open-palm gesture triggers an unload sequence, a closed-fist resumes operation, and the predicted path is rendered as a green-to-yellow trail 2 seconds ahead.

The whole project — simulation, vision pipeline, state machine, PyQt6 GUI, and tests — was built with Claude Code as the primary development partner, using conventional commits and pytest from day one.

Key Features

Circular trajectory tracking

End-effector draws a 15 cm radius circle at 0.3 m height using damped least-squares differential IK with a fixed downward orientation.

2-second trajectory preview

Twenty semi-transparent spheres show the predicted path ahead of the arm, making the robot's immediate intent visible to the operator.

MediaPipe gesture arbitration

Open-palm latches an unload-and-return sequence; closed-fist resumes from IDLE. A 5-frame hysteresis filter keeps noisy detections from flipping the state.

State machine + safety

RUNNING → UNLOADING → RETURNING_HOME → IDLE, with an independent Esc emergency-stop channel that freezes the state machine instantly.

Tech Stack

Python 3.12 MuJoCo mujoco_menagerie MediaPipe Hands OpenCV PyQt6 NumPy PyYAML pytest

Implementation Highlights

  • MjSpec assembly: UR5e and Robotiq 2F-85 are merged into a single nq=12, nu=7 model at runtime via the Python MjSpec API.
  • Damped least-squares IK: Pure NumPy Jacobian pseudoinverse controls position while keeping the gripper pointing downward.
  • Hysteresis filtering: 5-frame debounce (~165 ms) prevents gesture jitter from interrupting the task.
  • Off-screen rendering: MuJoCo's renderer is fed through a numpy-to-QImage adapter for the live simulation view.
  • Config-driven: Trajectory, IK, gripper, camera, gesture, and UI parameters are all tunable in configs/demo.yaml.
